# Changelog

## 1.3.4

- Added ImageGen long-tail latency and retry policy.
- Clarified that quiet waits, empty polling responses, and no intermediate progress are not failures.
- Added a minimum 6-minute wait before treating one ImageGen request as stalled.
- Added strict retry limits: original request plus at most one retry per image.
- Prohibited repeated identical prompt submissions while a previous request is still pending.
- Added status classification for `pending_empty_poll`, `completed`, `explicit_error`, `active_termination`, and `skipped_after_retry_limit`.
- Expanded ImageGen execution logging to capture attempt count, elapsed duration, error type, active termination, retry usage, and final action.
- Added guidance to continue the remaining image set after a failed image instead of blocking the whole set.
- Added API/async generation guidance for large batches.

## 1.3.3

- Added script validation for high-risk identical-background wording.
- Added script validation for repeated Main Visual Evidence Type / Visual Task values in generated planning tables.
- Added stronger Visual Anchor / Visual Variation separation.
- Added Image Style Distribution rules.
- Added anti-homogenization requirements.
- Updated planning table template with Visual Variation Role, Purchase Question Answered, and Main Visual Evidence Type.
- Updated prompt template with Visual Variation Role and anti-repeated-background requirements.
